# Software project roadmap

Use this phased sequence when the user wants to turn the workflow into a website.

## Phase 1: logic and schema
- finalize intake fields
- finalize assessment structure
- finalize interpretation rules
- finalize goal-generation rules
- define entities and example JSON

## Phase 2: backend prototype
- implement learner CRUD
- implement baseline result storage
- implement goal and weekly target storage
- implement weekly progress logging
- add computed weekly summary endpoint

## Phase 3: frontend prototype
- intake page
- assessment entry page
- learner profile page
- goals page
- weekly logging page
- analytics dashboard
- prompt generator page

## Phase 4: polish
- optional excel export
- printable parent summary
- authentication
- attachments and evidence uploads
